Fajtái

  • Böngésző (JS)

    • JavaScript könnyű értelmezésű ( vagy éppen időben fordított ) programozási nyelv a első osztályú funkciók. Noha a legismertebb a weboldalak szkriptnyelvének, sok nem böngésző környezet is használja, például Node.js, Apache CouchDB és Adobe Acrobat. JavaScript a prototípus-alapú, többparadigma, egyszálú, dinamikus nyelv, az objektum-orientált, kötelező és deklaratív ( pl. funkcionális programozás ) stílusok.

    • Ez a szakasz magának a JavaScript nyelvnek szól, nem pedig a weboldalakra vagy más gazdagépekre jellemző részekre. Információ a következőkről API-k amelyek a weboldalakra vonatkoznak, kérjük, olvassa el Webes API-k és DOM.

  • NodeJS

    • Node.js egy platformok közötti, nyílt forráskódú kiszolgálói környezet, amely képes futni Windows, Linux, Unix, MacOS, és még sok más. A Node.js a back-end JavaScript futási idő környezet, fut a V8 JavaScript motor, és végrehajtja a JavaScript kódot a webböngésző.

    • A Node.js lehetővé teszi a fejlesztőknek, hogy a JavaScript használatával parancssori eszközöket írjanak szerveroldali szkriptek. A JavaScript kód futtatásának képességét a kiszolgálón gyakran használják a generáláshoz dinamikus weboldal tartalom az oldal elküldése előtt a felhasználó böngészőjébe. Következésképpen a Node.js egy "JavaScript mindenhol" paradigmát képvisel, egyesítő web-alkalmazás fejlesztés egyetlen programozási nyelv körül, szemben a különböző nyelvek használatával a szerver és az ügyféloldali programozásban.

  • Deno

    • Deno /diːnʊ/ egy futási idő mert JavaScript, TypeScript, és WebAssembly ez a V8 JavaScript motor és a Rozsdás programozási nyelv. Deno társalkotója volt Ryan Dahl, aki szintén teremtett Node.js.

    • Deno kifejezetten vállalja mind a futásidejű, mind a csomagkezelő egy végrehajtható, ahelyett, hogy külön kellene csomagkezelő program.

  • Bun

    • A Bun egy sokoldalú eszközkészlet a JavaScript és a TypeScript alkalmazásokhoz. Egy végrehajtható néven szállítja bun​.

    • A lényege a Bun futásidejű, egy gyors JavaScript futási idő, amelyet a Node.js beépítésének helyettesítésére terveztek. Zig-ben írták, és a JavaScriptCore táplálja a motorháztető alatt, drámai módon csökkentve az indulási időket és a memória használatát.
